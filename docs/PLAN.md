## Решения по уточняющим вопросам

| Вопрос | Решение |
|--------|--------|
| Splash | Показывать **только при первом визите** (любая страница). Запоминать выбор в `localStorage` (ключ напр. `ais_skip_splash`). |
| Языки | **RU, EN, UZ** — мультиязычность с переключателем. |
| Формы | **Капча** (honeypot + опционально простая капча). Отправка в **Telegram-бот** — позже (пока API route-заглушка). |
| Контент | **Статичный** (в коде / JSON / MDX). |
| Цвета | См. раздел «Цветовая палитра» ниже. |
| Команда | **Аватарки и заглушки** (placeholder-фото, имя/роль). |
| Калькулятор | **Да.** Параметры: тип объекта, сроки, диапазон, лимиты сообщений/ответов, добавление админ-панели и дашборда для анализа. |

## Цветовая палитра (рекомендация)

### Текущее состояние

- **globals.css:** тёмный фон (#030303, #0A0A0A) + золотой акцент (#D4A853, #E8B86D) — премиум, B2B.
- **Services.tsx:** секция в сине-фиолетовой гамме (#1A1F3A, #00D4FF, #B829FF) — другой визуальный язык.
- **tailwind.config.ts:** primary #3B82F6 не совпадает с CSS-переменными.

### Рекомендация (клиенты + сеньор-оценка)

**1. Единая база — тёмный + золотой (как в globals.css)**  
Оставляем как основу: ассоциируется с надёжностью и премиальностью, подходит для B2B. Для клиентов — узнаваемый, «не кислотный» стиль.

**2. Унифицировать все секции под CSS-переменные**  
- Убрать хардкод сине-фиолетовых градиентов в Services (или перевести их в переменные «акцент по контексту»).
- Карточки услуг: градиенты иконок можно оставить брендовые (WhatsApp зелёный, Telegram голубой и т.д.), но **фон секции** — `var(--background)` / `var(--background-secondary)`, текст — `var(--foreground)` / `var(--muted-foreground)`.

**3. Расширить палитру для разработки и семантики**

В `globals.css` добавить (и при необходимости продублировать в Tailwind):

```css
/* Семантика для клиентов и a11y */
--success: #22C55E;      /* успех, подтверждение */
--warning: #EAB308;      /* предупреждение */
--error: #EF4444;        /* ошибка (уже есть destructive) */
--info: #3B82F6;         /* информация, ссылки в тексте */

/* Шкала нейтралей для консистентности (для сеньор-оценки) */
--neutral-50: #FAFAFA;
--neutral-100: #F5F5F5;
--neutral-200: #E5E5E5;
--neutral-500: #737373;
--neutral-700: #404040;
--neutral-900: #171717;
```

**4. Tailwind**  
В `tailwind.config.ts` подключать цвета из CSS-переменных (например через `var(--primary)` и т.д.), чтобы не дублировать hex и чтобы одна правка в `globals.css` применялась везде.

**Итог:**  
- Для клиентов: один стиль (тёмный + золотой), без резких контрастов между секциями.  
- Для сеньор-оценки: один источник правды (CSS variables), семантика (success/warning/error/info), контраст под a11y (WCAG AA по возможности).

## i18n (RU / EN / UZ)

- **Подход:** Next.js 14 App Router + `next-intl` (или аналог с поддержкой App Router).
- **Структура URL:** `/[locale]/...` (например `/ru/uslugi`, `/en/services`, `/uz/xizmatlar`).
- **Хранение переводов:** JSON в `messages/ru.json`, `messages/en.json`, `messages/uz.json` (или `src/i18n/`).
- **Определение локали:** по префиксу пути; fallback — RU. Опционально — сохранение выбора в cookie.
- **Splash, формы, калькулятор, навигация:** все строки через ключи переводов.

## Формы и капча

- **Валидация:** React Hook Form + Zod (уже в проекте).
- **Капча:**  
  - Обязательно: **honeypot** (скрытое поле, не заполняемое пользователем).  
  - Дополнительно: простая капча (например выбор картинки или вопрос), при необходимости — интеграция с сервисом (например Turnstile) позже.
- **Отправка:** API route `POST /api/contact` (или аналогичный). Внутри — проверка honeypot, валидация, затем вызов Telegram Bot API (реализация — позже). Пока возвращать 200 и логировать в консоль.

## Калькулятор стоимости — спецификация

| Параметр | Описание | Варианты / тип |
|----------|----------|-----------------|
| Тип объекта | Категория проекта | Выбор: например «Чат-бот», «CRM-интеграция», «Дашборд», «Комплексное решение» и т.д. |
| Сроки | Желаемый дедлайн | Выбор диапазона или фиксированные опции (2 нед, 1 мес, 2 мес, 3+ мес). |
| Диапазон | Масштаб / объём | Упрощённо: «Старт», «Бизнес», «Энтерпрайз» или слайдер/диапазон. |
| Лимит сообщений | В месяц (или иной период) | Число или выбор из пресетов (1k, 5k, 10k, 50k, без лимита). |
| Лимит ответов | В месяц (или иной период) | Аналогично. |
| Админ-панель | Нужна ли админка | Да / Нет (чекбокс или переключатель). |
| Дашборд для анализа | Аналитика и отчёты | Да / Нет (чекбокс или переключатель). |

**Выход:**  
- Диапазон стоимости (мин–макс в руб/долл) на основе правил (можно захардкодить в конфиге/константах).  
- Опционально: кнопка «Получить КП» → форма контакта + сохранение выбранных опций для последующей отправки в ТГ.

## Порядок разработки (напоминание)

| Фаза | Задачи |
|------|--------|
| **1. Фундамент** | Splash (первый визит, localStorage), роутинг всех страниц, i18n (next-intl + 3 локали, базовая навигация). |
| **2. Контент и SEO** | Заполнение страниц, metadata по страницам и локалям, sitemap с `[locale]`. |
| **3. Анимации и UX** | ScrollTrigger, переходы, 3D hover на карточках, счётчики, оптимизация (lazy, reduced motion). |
| **4. Формы и полировка** | Формы с honeypot и капчей, API route, калькулятор (UI + логика диапазона), a11y, Lighthouse. |

Дополнительно: блог, PDF КП, интеграция отправки в ТГ — после фазы 4.

## Риски и узкие места (кратко)

- **i18n:** Роутинг и middleware next-intl нужно настроить с первого шага, чтобы не переделывать ссылки.
- **Калькулятор:** Чётко зафиксировать правила расчёта (таблица или конфиг), иначе бесконечные правки.
- **Цвета:** Привести Services и tailwind к одной палитре (CSS variables) в рамках одной итерации, чтобы не плодить расхождения.
